ТЕХНИЧЕСКОЕ ЗАДАНИЕ на разработку программы «DataProcessor»

# ВВЕДЕНИЕ

Наименование программы: «DataProcessor».

Программа представляет собой веб-приложение для управления задачами анализа HDL проектов с интеграцией системы LibreLane. 
Область применения: автоматизация процессов анализа проектов на языках описания аппаратуры (Verilog, SystemVerilog) в научно-исследовательских и производственных организациях, занимающихся разработкой электронных устройств.

Объект применения: вычислительные комплексы и рабочие станции инженеров-разработчиков электроники.

# ОСНОВАНИЯ ДЛЯ РАЗРАБОТКИ

Условное обозначение темы: «DataProcessor»

# НАЗНАЧЕНИЕ РАЗРАБОТКИ

Функциональное назначение: обеспечение веб-интерфейса для загрузки конфигурационных и исходных файлов HDL проектов, управления задачами анализа через систему LibreLane, отслеживания хода выполнения и получения результатов анализа.

Эксплуатационное назначение: программа предназначена для использования инженерами-разработчиками и исследователями в процессе разработки и верификации HDL проектов

# ТРЕБОВАНИЯ К ПРОГРАММНОМУ ИЗДЕЛИЮ

## Требования к функциональным характеристикам

Программа должна обеспечивать выполнение следующих функций.

Аутентификация и управление доступом:

- Аутентификация пользователей через email с использованием механизма токенов
- Защита от несанкционированного доступа к задачам

Управление файлами:

- Загрузка конфигурационных файлов (форматы: JSON, YAML, YML, CONF)
- Загрузка исходных кодов HDL (форматы: V, VH, SV)
- Валидация загруженных файлов по расширениям и содержимому
- Максимальный размер загружаемых файлов: 10 МБ

Управление задачами анализа:

- Создание задач (Run) на анализ загруженных файлов
- Интеграция с системой LibreLane для выполнения анализа
- Отслеживание статуса выполнения в реальном времени
- Просмотр детальных логов выполнения
- Возможность отмены выполняемой задачи

Пользовательский интерфейс:

- Веб-интерфейс на русском языке
- Отображение прогресса выполнения задач

## Требования к надежности

Обеспечение устойчивого функционирования:

- Время непрерывной работы без сбоев: не менее 2-уя месяцев
- Контроль целостности данных при загрузке файлов
- Валидация входных данных перед обработкой

Восстановление после сбоев:

- Время восстановления после программного сбоя: не более 5 минут
- Автоматическое сохранение состояния задач в базе данных
- Резервное копирование конфигурационных данных

Обработка ошибок

- Детальное логирование всех операций
- Защита от некорректных входных данных
- Гибкая система обработки исключительных ситуаций

## Требования к составу и параметрам технических средств

Минимальная конфигурация:

- Процессор: x86-64 архитектура, 2 ядра, тактовая частота от 2 ГГц
- Оперативная память: 4 ГБ
- Дисковое пространство: 10 ГБ
- Сетевой адаптер: Ethernet 100/1000 Мбит/с
- Устройства ввода-вывода: клавиатура, монитор (разрешение не менее 1280×720)

Рекомендуемая конфигурация:

- Процессор: 4+ ядер, тактовая частота от 3 ГГц
- Оперативная память: 8 ГБ
- Дисковое пространство: 50 ГБ (SSD)
- Сетевой адаптер: Gigabit Ethernet
- Резервное питание: ИБП

## Требования к информационной и программной совместимости

Информационная совместимость:

- Входные форматы: JSON, YAML, Verilog (.v), SystemVerilog (.sv, .vh)
- Выходные форматы: TAR, TAR.GZ, текстовые логи
- Кодирование данных: UTF-8

Программная совместимость:

- Операционные системы: Windows 10/11, Linux (Ubuntu 18.04+, CentOS 7+), macOS 10.15+
- Веб-браузеры: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- Система управления базами данных: SQLite 3.35+, PostgreSQL 12+
- Python 3.8+

Внешние интерфейсы:

- REST API для интеграции с системой LibreLane
- SMTP протокол для отправки email уведомлений
- HTTP/HTTPS для веб-доступа

Защита информации:

- Аутентификация пользователей
- Контроль доступа к задачам
- Шифрование передаваемых данных (HTTPS)
- Защита от SQL-инъекций и XSS атак

## Специальные требования

Производительность:

- Время отклика веб-интерфейса: не более 2 секунд
- Время загрузки файлов: зависит от размера, но не более 30 секунд для 10 МБ
- Поддержка одновременной работы 10+ пользователей

Безопасность:

- Использование безопасных алгоритмов шифрования
- Регулярное обновление зависимостей
- Аудит безопасности кода
